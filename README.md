# hugo-src

Source files for building the personal website using Hugo. This folder contains the site configuration, content, theme, static assets and build resources used to generate the published site in the `public/` folder.

## Quick Start

- **Prerequisites:** Install Hugo (see https://gohugo.io/getting-started/installing/). Git is helpful for versioning. On Windows you can use PowerShell, Git Bash, or WSL to run the commands below.
- **Development server:** Run the site locally for development:

```bash
hugo server -D
```

- **Build production site:** Generate the static site into the `public/` directory:

```bash
hugo
```

- **Helpful script:** There is a `run_hugo.sh` script at the repository root — use it in a Unix-like shell (Git Bash / WSL). On Windows PowerShell run the Hugo commands above directly.

## Project Layout (important folders)

- `config.toml` — site configuration and theme settings.
- `content/` — markdown pages and posts (edit or add `.md` files here).
- `themes/` — bundled theme(s). This site uses the `bingo` theme located at `themes/bingo`.
- `layouts/` and `themes/*/layouts` — custom templates and partials.
- `resources/` — Hugo-processed assets (generated during builds).
- `public/` — generated output (the built website). This folder may be committed or generated by CI.

## Editing Content

- Add or edit Markdown files under `content/` (for example, `content/about.en.md`). Use front matter (TOML/YAML) to set title, date, taxonomy terms, and draft status.
- To preview drafts locally, run `hugo server -D`.
- Do not add a file named `files.md` under `content/` as it conflicts with the `/files/` URL path used for static assets.

## Adding Images and Assets

- Upload images and static files to the `public-files/` folder in the Fastmail file storage. Then reference them in your content using the URL format `https://files.yurioku.com/your-image.jpg`.

## Theme and Styling

- The site uses the `bingo` theme inside `themes/bingo`. To change theme behavior, edit templates under `themes/bingo/layouts` or override them by adding files to the top-level `layouts/` folder.

## Deploying

- Build the site (`hugo`) and publish the contents of `public/` to your hosting provider (GitHub Pages).
- Typical CI flow: checkout source, run `hugo`, and deploy the `public/` directory.

## Notes

- If `public/` is committed to this repo, you can regenerate it locally and inspect changes before pushing.
